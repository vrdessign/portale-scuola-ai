FASE 1 — CREA IL REPOSITORY GITHUB
══════════════════════════════════════════════════════════════

PASSO 1 — Vai su github.com (loggato come vrdessign)

PASSO 2 — Clicca "+" in alto a destra → "New repository"

PASSO 3 — Compila il form:
  Repository name:  portale-scuola-ai
  Description:      EduRAG — Dual RAG Educativo (D-RAG-E) architecture
                    for sovereign AI in national education systems
  Visibility:       ✅ Public
  Initialize:       ✅ spunta "Add a README file" (lo sovrascriveremo)
  .gitignore:       None (lo carichiamo noi)
  License:          None (lo gestiamo noi tramite CITATION.cff)

PASSO 4 — Clicca "Create repository"

──────────────────────────────────────────────────────────────
PASSO 5 — Carica i file (metodo drag & drop, nessun terminale)

  5a. Clicca "uploading an existing file" (link nella pagina vuota)
      OPPURE: tasto "Add file" → "Upload files"

  5b. Carica questi file dalla cartella del progetto v2:
      ✅ main.py
      ✅ config.py
      ✅ requirements.txt
      ✅ docker-compose.yml
      ✅ Dockerfile
      ✅ .env.example
      ✅ README.md          (quello che hai scaricato da qui)
      ✅ CITATION.cff
      ✅ .zenodo.json
      ✅ .gitignore

  5c. Per le cartelle app/ e data/:
      GitHub non accetta cartelle vuote via drag & drop.
      Usa questo metodo per ogni file dentro app/:

      → Clicca "Add file" → "Create new file"
      → Nel campo nome scrivi:  app/rag/dual_rag.py
      → Incolla il contenuto del file
      → Clicca "Commit new file"
      → Ripeti per ogni file .py del progetto

  5d. File da creare in questo modo (uno per uno):
      app/__init__.py                    (file vuoto)
      app/rag/__init__.py                (file vuoto)
      app/rag/dual_rag.py
      app/rag/embeddings.py
      app/rag/vectorstore.py
      app/rag/llm_local.py
      app/rag/document_processor.py
      app/api/__init__.py                (file vuoto)
      app/api/routes/__init__.py         (file vuoto)
      app/api/routes/studente.py
      app/api/routes/docente.py
      app/api/routes/dirigente.py
      app/api/routes/admin.py
      app/auth/__init__.py               (file vuoto)
      app/auth/utils.py
      app/auth/routes.py
      app/database/__init__.py           (file vuoto)
      app/database/models.py
      app/database/session.py
      data/ministeriale/README.md
      data/uploads/.gitkeep             (file vuoto)
      .github/workflows/release.yml

PASSO 6 — Verifica che il repository sia visibile e il README
          si mostri correttamente nella pagina principale.

FASE 2 — COLLEGA ZENODO A GITHUB
══════════════════════════════════════════════════════════════

PASSO 7 — Vai su zenodo.org

PASSO 8 — Clicca "Log in" → "Login with GitHub"
          (NON creare un account separato con email)
          Autorizza Zenodo ad accedere al tuo account GitHub.

PASSO 9 — In Zenodo: clicca sul tuo nome in alto a destra
          → "GitHub" (nella lista a sinistra del profilo)

PASSO 10 — Trova "portale-scuola-ai" nella lista dei tuoi repository
           → Attiva l'interruttore (ON)
           Zenodo ora monitora le release di quel repository.
